# `results/` — Every AI response and every judge score

**What's in this folder:** The experiment's output side. For each subject and each condition, the model's response and every judge's 1-5 score of that response.

Reading `docs/FILE_NAMING.md` first is strongly recommended. The naming here is terse on purpose (because there are thousands of files) and the schema is documented once there.

## Contents

**Top-level aggregate files:**

- `RESULTS_S113.json`: The consolidated results aggregate from session S113. This is what `docs/DATA_REFERENCE.md` was generated from. A new `RESULTS_S114.json` will land once the 5-judge primary recompute finishes.
- `*_analysis.json` and `*_manifest.json` (letta, mem0, supermemory, zep): Per-provider analysis summaries and run manifests (SDK versions, timestamps).

**Per-subject folders (one per subject):**

- `hamerton/`: The primary subject. Full set of conditions.
- `franklin/`: Known-figure control.
- `franklin_obscure/`: Exploratory Franklin cross-corpus test.
- `global_augustine/`, `global_babur/`, `global_bernal_diaz/`, `global_cellini/`, `global_ebers/`, `global_equiano/`, `global_fukuzawa/`, `global_keckley/`, `global_rousseau/`, `global_seacole/`, `global_sunity_devee/`, `global_yung_wing/`, `global_zitkala_sa/`: The 13 global subjects used in the main gradient analysis.

**Special folders (leading underscore means "not the main per-subject tree"):**

- `_tier2/`: Circularity-control reruns where the test battery was generated by a different model (GPT-5.4) and rescored by non-Haiku response models. Tests whether the spec advantage persists when the question-writing model changes.
- `_wrong_spec_v2/`: The wrong-spec control. Each subject is run with a different (randomly deranged) subject's spec, to test whether a correct spec matters or any structured prompt would do.
- `_s114_backfills/`: Per-cell rejudging from S114 that filled gaps in the original judge coverage.

**Other:**

- `judge_calibration/`: Calibration diagnostics per judge (see its README).
- `multimodel/`: Hamerton responses generated by non-Haiku response models as a cross-model sanity check.

## How naming works here

Inside each per-subject folder, filenames follow this pattern (fully documented in `docs/FILE_NAMING.md`):

- `<system>_results.json`: Model responses for that system's conditions.
- `<system>_judgments_<judge>.json`: One judge's 1-5 scores for that system's responses.
- `<system>_judgments_merged.json`: All judges' scores merged into one file.
- `<system>_fullpipeline_*.json`: Native configuration (the system does its own extraction).
- `<system>_ingestion.json`, `<system>_retrieval.json`: What the system stored and what it retrieved per question.
- `baselayer_*.json`: Base Layer's substrate (MiniLM + ChromaDB retrieval). Base Layer is not a memory system; it is the spec layer. The retrieval files are just a local retrieval floor for comparison.
- `c8_c9_*.json`: The raw-corpus condition (C8) and raw-corpus-plus-spec condition (C9).
- `results.json` vs `results_v2.json`: The `_v2` version is the one the paper uses; the original is kept for audit.
- Trailing `.rl_backup` or `.brief_only_backup` files are pre-rerun backups; do not use them as primary data.

## Where these files come from / go to

Inputs: prompts and batteries from `data/`. Outputs: consolidated into `RESULTS_S113.json` (and soon `RESULTS_S114.json`), then summarized in `docs/DATA_REFERENCE.md`, then cited in the paper.

## Caveats worth knowing

- Parse failures are encoded as `{"score": 0, "parse_failure": true}` in judgment files. Aggregating without filtering these will skew averages. The 5-judge primary recompute script handles this correctly.
- Supermemory native ingestion failed for 4 subjects (Babur, Bernal Diaz, Cellini, Rousseau). Their `C1_supermemory_fp` responses are empty. Ingestion issue, not a judge issue.
- The supermemory paid-tier rerun (files matching `supermemory_fullpipeline_*` beyond the bare `_retrieval.json`) is intentionally scoped to aggregate analysis; per-judge files exist for Augustine + Keckley only. The other 11 global subjects retain only `supermemory_fullpipeline_retrieval.json` from this rerun.
- The 7-judge aggregate in `RESULTS_S113.json` is a flat all-judges-available aggregate. The 5-judge primary recompute is the reporting default going forward.
