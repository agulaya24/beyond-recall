# `results/_s114_backfills/` — Per-cell rejudging from session S114

**What's in this folder:** Judge scores rerun in session S114 to fill gaps in the original judge coverage. During the main runs, OpenAI and Google rate limits caused missing scores on specific subject × condition × judge cells. These files fill those holes.

## Why the backfills exist

Twelve of the thirteen global subjects had sustained 429 rate-limit errors from OpenAI and Google during the original Base Layer judgment runs. That left some cells without complete per-judge coverage. S114 reran just the missing cells so the 5-judge primary panel is complete across all subjects.

## Contents

One file per (subject × condition × judge) cell that needed a backfill. Filenames follow:

`<subject>__<condition>__<judge>.json`

For example, `global_augustine__C1_baselayer__gemini_flash.json` means "Augustine, Base Layer retrieval only condition, scored by Gemini Flash".

Observed subjects in this folder: `global_augustine`, `global_babur`. Conditions covered: `C1_baselayer`, `C3_baselayer`, `C1_supermemory`, `C3_supermemory`, `C8_raw_corpus`. Judges: `gemini_flash`, `gpt4o`, `gpt54`.

## How naming works here

Double-underscore (`__`) separates the three components: subject, condition, judge. This is intentionally different from the single-underscore inside each component name (e.g., `C3_baselayer`, `raw_corpus`). The format makes parsing straightforward.

## Where these files come from / go to

Generated by the S114 rejudging scripts in `scripts/` (for example, backfill runners that loop over missing cells). Outputs merge into the 5-judge primary recompute at `docs/research/recompute_5judge_primary.md` and will flow into the forthcoming `RESULTS_S114.json`.

## Caveats worth knowing

- These files are partial by design. They are backfills, not full reruns. Do not treat this folder as a second, independent run.
- When aggregating, these cells should be merged into the main per-subject folder's `<system>_judgments_<judge>.json`, not counted separately.
- Hamerton's three-judge backfill (Sonnet, Opus, GPT-4o) lives inside `results/hamerton/` as `sonnet_judgments.json`, `opus_judgments.json`, `gpt4o_judgments.json`, not here.
