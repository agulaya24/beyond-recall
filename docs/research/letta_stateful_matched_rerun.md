# Letta Stateful vs BL C2a — Matched-Battery, Same-Spec Rerun (Ebers, Babur)

**Purpose.** Resolve three pre-launch concerns raised in `letta_stateful_deep_read.md` about the §1.3 / §4.3.1 Letta stateful-agent comparison: (1) verify the Babur block ceiling claim, (2) address the alleged anonymization confound, (3) eliminate the 39/39 question-text mismatch between the BL C2a run and the Letta stateful run on Ebers and Babur.

**TL;DR.**
- **Ceiling.** The Babur block saturates around **~333K chars**: last successful ingestion at T220 with block size **332,585 chars**; first `400 Bad Request` at T221. Block grew to **335,349 chars** by the post-loop final fetch. The paper's "333,000-character per-message API ceiling" is directionally correct but imprecise; the effective ceiling is a combined request-body size (existing block + new chunk). Block metadata declares `limit=100,000` but that cap is not enforced.
- **Anonymization claim is wrong.** The spec actually used in the paper's C2a run for Ebers and Babur is the NAMED spec generated by `run_global_subjects.py` (not the anonymized `spec_production.md`). The original comparison was never anonymized-vs-named.
- **Matched-battery rerun (with the paper's named spec).** BL C2a still trails Letta stateful at matched conditions, but the gap shrinks substantially:
  - Ebers: paper Δ +1.21 → matched Δ **+0.75** (7-judge, aligned with paper's 3.00 column); **+1.04** on strictly-matched non-Gemini 4-judge
  - Babur: paper Δ +0.57 → matched Δ **+0.29** (7-judge, aligned with paper's 2.73 column); **+0.54** on strictly-matched non-Gemini 4-judge
- **Judge-set asymmetry.** GPT-5.4 returned HTTP 400 on all 40 Letta stateful calls but worked on all 40 BL-C2a-named calls. The "7-judge" headline is actually Letta-6-judge vs BL-7-judge. GPT-5.4 scored BL slightly lower than the calibrated core, so the 7-judge number slightly understates the gap. Non-Gemini 4-judge is the apples-to-apples comparison.
- **Honest interpretation.** The original Letta-beats-BL claim is still supported after resolving the battery + anonymization confounds. The gap is roughly half the paper's published Δ on the aligned 7-judge metric; roughly 85% of the paper's published Δ on the strictly-matched 4-judge metric. The remainder comes from Letta's block carrying corpus-derived narrative content (which Haiku can cite on held-out questions about specific events) while the BL behavioral spec carries only behavioral scaffolding. On low-baseline subjects, Haiku without corpus-derived anchoring scores at baseline-adjacent levels even with a strong spec.

## Part 2 — The anonymization premise is wrong

The pre-launch concern was that the BL spec is authored with anonymization ("this person" / "the individual") while Letta's block names the subject, and the resulting refusal pattern on low-baseline subjects could be an anonymization artifact rather than a representational gap.

**Finding:** The spec actually used in the paper's C2a run for Ebers and Babur (and other global subjects) is NOT the anonymized `data/global_subjects/{subject}/spec_production.md`. It is the NAMED spec generated by `memory-study-repo/scripts/run_global_subjects.py` at runtime.

Definitive evidence from `run_global_subjects.py:260-272`:

```python
spec_prompt = (f'You are creating a behavioral specification for {display}. '
               'Based on these facts, write a compressed document describing how this person '
               'thinks, decides, and acts. Focus on: decision patterns, values, '
               'conflict resolution, learning style, risk tolerance, relationship dynamics. '
               'Write in third person. Be specific and behavioral, not biographical.\n\n'
               f'Facts:\n{facts_text}')
# ... Haiku call produces spec ...
with open(os.path.join(outdir, 'spec.md'), 'w', encoding='utf-8') as f:
    f.write(spec)
```

The saved `spec.md` (7,259 chars for Ebers, 6,784 for Babur) opens with `# BEHAVIORAL SPECIFICATION: GEORG EBERS` and names the subject throughout. Sample from `results/global_ebers/spec.md`:

> Ebers demonstrates a deliberate, intellectually-grounded approach to major life decisions. He commits to long-term pursuits with sustained focus—spending twelve years between his first and second novels...

And `run_global_subjects.py:303-309` shows this spec is what loads into `C2a_full_spec`:

```python
if spec:
    try:
        r = gen_response(q_text,
            'The following is a behavioral specification describing your user.\n\n'
            '=== BEHAVIORAL SPECIFICATION ===\n' + spec)
        q_result['responses']['C2a_spec'] = r
```

**Implication.** The original BL-C2a and Letta-stateful comparisons on Ebers and Babur are already named-against-named. The refusal pattern documented in `letta_stateful_deep_read.md` §2.2.1 exists even with the named spec — it's driven by Haiku's lack of pretraining knowledge of the subject combined with the spec's epistemic-honesty axioms, not by name redaction. This rerun therefore uses the same named spec (the paper's `results/global_{subject}/spec.md`) for the matched-battery test. Part 2's "de-anonymization" step reduces to a no-op: we copied the paper's actual C2a spec verbatim.

Copies at `_letta_rerun/ebers_spec_named.md` and `_letta_rerun/babur_spec_named.md` are byte-identical to `results/global_ebers/spec.md` and `results/global_babur/spec.md`.

**Separate note.** The anonymized `spec_production.md` files (the full 4-layer anchors+core+predictions+brief, ~50K chars) exist at `data/global_subjects/{subject}/spec_production.md` and ARE used by `run_baselayer_condition.py` for the Base Layer full-stack memory-system condition (C1_BaseLayer / C3_BaseLayer), not for the C2a gradient column. ## Interpretation

### The original Letta-beats-BL claim survives, with reduced magnitude

After resolving:
- ✓ the battery mismatch (matched 40/40),
- ✓ the anonymization assumption (always was named-against-named),
- ✓ gpt54 parse-failure asymmetry (gpt54 worked on rerun, failed on Letta stateful — inflates rerun slightly relative to original),

the Letta stateful block still scores higher than the BL C2a spec on both Ebers (+0.75) and Babur (+0.29) at matched response model. The effect is roughly half as large as the paper's published Δ (Ebers +1.21, Babur +0.57), but the direction is unchanged and robust across judge subsets.

### The remaining gap is about content, not anonymization or refusal

Letta's stateful memory block for Ebers (68K chars) and Babur (335K chars) consists of narrative summaries lifted from the first-half corpus — the ingested material is paraphrased into the block but remains concrete. Haiku, when served this block as context, has corpus-derived material to cite and reason from on questions about specific held-out events.

The BL C2a spec for Ebers (7,259 chars) and Babur (6,784 chars) is 10-50× smaller than Letta's block and contains only behavioral scaffolding — no historical content, no specific episodes, no named relationships beyond what's in the compressed behavior description. On questions where the held-out passage describes a specific event, a specific named person's reaction, or a specific material reward, the BL spec can frame an engaged and coherent answer but often misses the specific content.

This is consistent with the paper's §4.2 Hamerton compression result: *"C4 (all facts, no spec) at 7,723 tokens scores 2.53, while C2a (spec alone) at 7,320 tokens scores 3.04"*. The BL spec IS the right format when the response model has some anchored content to compose with. On low-baseline subjects where the model has near-zero pretraining anchoring (C5 Ebers = 1.04, C5 Babur = 1.98), the spec alone cannot fully substitute for the corpus-derived narrative a Letta block carries. This is why the paper's C4a (facts + spec) consistently outperforms C2a (spec alone) on low-baseline subjects.

### Paper-level implications

1. **§4.3.1 claim is directionally correct, magnitude should be reduced.** Letta's stateful path produces a representation that outscores a behavioral spec at matched response model — but on Ebers/Babur the matched-battery gap is about half the magnitude the paper currently implies via the mismatched-battery numbers.

2. **The "architectural convergence" frame is still right.** Both Letta's block and the BL spec are interpretive compressed representations; they exhibit different information profiles (narrative-summary vs. behavioral-scaffold) and different fall-off patterns at scale (Babur: Letta's block hit a server ceiling at ~333K; BL's compose step produces a 6,784-char spec regardless of corpus size). The paper's framing of this as a frontier question rather than a superiority claim remains accurate.

3. **The corpus-content axis is what distinguishes the two representations on low-baseline subjects.** The BL spec's ~7K chars do not carry ingested narrative content. Letta's ~68K-335K blocks do. A useful follow-up experiment: serve Haiku with the BL spec AND the same training corpus summarized into a narrative block (i.e., a BL-spec + BL-narrative-block composite). If this closes the matched-battery gap, the content axis is confirmed.

## Part 7 — 5-judge primary panel (GPT-5.4 fix, post-rerun)

**Context.** §4.7 in the paper draft reports Letta vs BL on a 7-judge aggregate because GPT-5.4 returned HTTP 400 on all 40 Letta stateful calls during the original rerun. The paper's canonical convention elsewhere is the **5-judge primary panel** (Haiku, Sonnet, Opus, GPT-4o, GPT-5.4; excludes Gemini Flash and Gemini Pro). This section reports §4.7's Letta vs BL Δ on the 5-judge primary panel after fixing the GPT-5.4 failure.

### Root cause of the GPT-5.4 HTTP 400

The original scoring script (`memory_system/data/experiments/memory_systems/judge_tier2.py::judge_gpt54`) sent `max_tokens=10` to the OpenAI Chat Completions endpoint. GPT-5.x rejects that parameter:

> `"Unsupported parameter: 'max_tokens' is not supported with this model. Use 'max_completion_tokens' instead."` (HTTP 400, `code=unsupported_parameter`)

The 400 fires before any inference runs, which is why every call failed identically regardless of payload size. Payload size was never the issue — the largest prompt in this battery is ~1.5K input tokens, well under any GPT-5.4 ceiling. The rerun script used for BL-C2a-named responses (`_letta_rerun/40_judge_responses.py`) already used `max_completion_tokens` and worked cleanly; only the original Letta stateful judge path was broken.

Reproduction in `_letta_rerun/diag_reproduce_gpt54.py`: with `max_tokens` → 400; with `max_completion_tokens=10` → 200 with a valid 1–5 score.

### Fix

Added a new scorer (`_letta_rerun/60_rerun_gpt54_letta.py`) that uses `max_completion_tokens=10` and writes both to the canonical memory_system paths (in-place overwrite of the 39/40/40 all-zero judgment files) and to traceability copies in `_letta_rerun/letta_{subject}_judgments_gpt54.json`. All 119 calls succeeded on the first pass (39 Hamerton + 40 Ebers + 40 Babur). Per-judge means from the fixed GPT-5.4 on the Letta block → Haiku condition: Hamerton 3.051, Ebers 2.975, Babur 2.325.
