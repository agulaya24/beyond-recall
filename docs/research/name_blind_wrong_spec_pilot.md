# Name-Blind Wrong-Spec Pilot — NOT RUN (Premise Empirically Falsified)

*Target: produce a defensible lower-bound on behaviorally-grounded wrong-spec detection, accounting for the "specs are named" caveat on the 60.6% figure in `wrong_spec_detection_analysis.md`.*

## TL;DR

The pilot was not executed. Pre-flight verification of the input data showed the premise of the task is empirically false: **the behavioral specifications used to compute the 60.6% explicit-detection rate are already name-blind.** Every `spec_production.md` file the C2c_wrong_spec condition loaded is fully anonymized to "this person" / "they" / "them" and contains zero instances of the source subject's name. The classifier's explicit-detection rate is therefore not an upper bound inflated by trivial name-mismatch; it is already a behaviorally-grounded detection rate, operating on content-only cues.

**Implication for the paper:** the Scope and Caveat paragraph in `wrong_spec_detection_analysis.md` (line 27) incorrectly asserts "Specs are named, not anonymized." That sentence must be corrected before launch. The actual scope caveat is narrower — identification happens via cultural/temporal/domain cues embedded in spec content (e.g., "Middle Passage", "Christian slaveholders", "provincial environment", "classical rhetoric"), not via names.

## What the Pilot Would Have Been

1. Pick 2-3 low-baseline subjects where wrong-spec v2 was most informative (candidates: Ebers, Keckley, Hamerton).
2. For each assigned wrong-spec, replace every instance of the source subject's name with a neutral placeholder.
3. Run Haiku with each anonymized wrong-spec against each target subject's 39 BP questions.
4. Classify responses using the existing 4-way classifier (`scripts/classify_wrong_spec_detection.py`).
5. Compare the name-blind detection rate against the 60.6% upper-bound.

## What Was Done Instead

Before running the pilot, the specs were inspected directly to confirm the presence of names that would need anonymizing. They are not present.

    return open(path, encoding='utf-8').read()

This loads `memory-study-repo/data/global_subjects/<subject>/spec_production.md` — the production spec for each of the 13 global subjects.

Anchors, core, and predictions layers follow the same convention.

### 2. For Hamerton (v1 Franklin's spec), the brief is 95% anonymized

Hamerton's wrong-spec is Franklin's `brief_v5_clean.md` (loaded by `run_full_study.py` line 48-50). Grep for `Franklin|Benjamin` in that file: **1 match**, line 33, in body text: "... raised in Boston by Josiah Franklin and Abiah Folger, the youngest son of the youngest son for five generations." This is a parental surname in a biographical clause, not a first-person label. The spec still refers to Franklin throughout as "he" / "him" / "his" in the injectable block.

The 88% explicit-detection rate on Hamerton-with-Franklin's-spec is therefore also not primarily driven by name-reading. Franklin is a famous figure whose content fingerprint (18th-century American printer/scientist/diplomat, electrical experiments, Royal Society medal, Deborah Read, Pennsylvania Gazette, Philadelphia) is trivially recognizable to any competent LLM.

### 3. Explicit-detection evidence quotes show content-based inference, not name-reading

From `wrong_spec_detection_raw.json`, classified-"explicit" examples for Augustine (wrong-spec source = Fukuzawa):

And from the original report's example quotes:

- Keckley (wrong-spec = Babur via the `run_global_rerun.py` mapping): *"I need to flag immediately: this specification is not about Elizabeth Keckley. This is a behavioral model of a 16th-century Central Asian military ruler (almost certainly Babur...The specification itself notes it's built from self-report"*
- Cellini: *"The specification you've provided describes a contemporary person with Dakota Sioux identity, shaped by missionary education separation, operating between two worlds with deep structural suspicion of institutions. Cellini was a 16th-century..."*

In every case the model is inferring the spec's subject from content (temporal markers, cultural domain, life events, epistemic architecture) and comparing it against what it knows about the question's target. Names are not the mechanism.

### 4. The specs do contain identifying content cues

Even though names are absent, specs contain domain-specific content that is easily matchable to a known person. Example: Equiano's `spec_production.md` (wrong-spec source for Ebers under the `run_global_rerun.py` mapping) contains:

- Line 114: *"Cultural relativism without structural critique — all practices become equally valid, including exploitative ones, with no basis for condemning the slave trade as a system."*
- Line 116: *"... condemning the slave trade while implicitly affirming European moral superiority as the standard of condemnation."*
- Line 183: *"... where Christian rhetoric is deployed by those who violate Christian teaching..."*
- Line 214: *"They were born free, enslaved by force, and experienced the Middle Passage as a captive."*

A model reading this spec and being asked about Georg Ebers will recognize the content is about a West African formerly-enslaved subject, not a 19th-century German Egyptologist. The explicit-detection rate of 54% on Ebers is driven by this content fingerprint, not by a name.

### 5. Two distinct wrong-spec runs exist; both use name-free specs

- **Run A** (generator: `run_global_rerun.py`, artifact: `results_v2.json`, classifier input for the 60.6%): uses `WRONG_SPEC_PAIRING` — a deterministic dict (e.g., ebers → equiano, keckley → babur). Loads `spec_production.md` files, which are the **fully anonymized** versions.
- **Run B** (generator: `run_wrong_spec_v2.py`, artifact: `wrong_spec_v2_results.json`, separate downstream analysis): uses a seed=42 random derangement (e.g., ebers → fukuzawa, keckley → augustine). Loads the 4 full-stack injectable blocks from `global_subject_environments/<subject>_memory/data/identity_layers/`, which are **also fully anonymized**.

Both mappings produce name-free wrong-spec system prompts. The 60.6% figure comes from Run A.

## Comparison to the 60.6% Upper-Bound Claim

The claim in `wrong_spec_detection_analysis.md`: *"A portion of 'explicit detection' is therefore triggered by trivial name-mismatch rather than deeper behavioral-mismatch recognition."*

This is not supported by the data. Names are absent. The correct framing is:

- The 60.6% is already a content-grounded detection rate.
- A subordinate concern remains: content cues in the specs (dates, places, cultural/religious domains, specific life events) can be recognized by a model with strong world knowledge. This is not name-mismatch; it is domain/era-mismatch.
- A genuinely content-stripped lower-bound experiment would require anonymizing not just names but also culturally-identifying terms. That is a different and harder experiment, and it would also be harder to interpret because stripping all domain markers eliminates exactly the behavioral content the spec is supposed to convey.

## Proposed Correction to `wrong_spec_detection_analysis.md`

Replace the "Scope and Caveat" paragraph (lines 26-30):

**Current:**
> **Specs are named, not anonymized.** The model sees a spec that opens with the name of a different historical figure (e.g., Augustine's question paired with a spec titled "Yung Wing"). A portion of "explicit detection" is therefore triggered by trivial name-mismatch rather than deeper behavioral-mismatch recognition. The classifier does not distinguish these. Treat the explicit rate as an upper bound on detection that is grounded in careful reading of the spec's behavioral content; it is a lower bound on detection of *any* sort (including trivial name mismatch). The gap matters for the paper's narrative.

**Proposed:**
> **Specs are name-free but contain domain cues.** The production specs used in the C2c_wrong_spec condition (`data/global_subjects/<subject>/spec_production.md`) refer to the subject exclusively as "this person" / "they" and contain no instances of the subject's name. Explicit detection is driven by content fingerprinting — models infer the spec's subject from temporal, cultural, and domain markers embedded in behavioral examples (e.g., "Middle Passage", "classical rhetoric", "provincial clan system"). The 60.6% rate is therefore a content-grounded detection rate, not a name-mismatch artifact. A subordinate limitation is that famous or culturally-distinctive subjects are easier to detect from content cues alone; this is visible in the per-subject variance (Hamerton 88% vs Bernal Diaz 21%).

## Follow-Up Options (Not Executed)

If a tighter lower-bound on purely-behavioral detection is wanted, the honest next experiment would strip cultural/temporal/domain markers from specs (not names — those are already gone). This is non-trivial because those markers often load-bear the behavioral patterns themselves (e.g., "calibrated discretion under feudal clan surveillance" cannot be stripped of "feudal" without losing the behavioral signal). Expect a meaningful drop in detection rate at the cost of spec legibility. Propose before running.
