## Critical issues must fix before submission

### 1. §3.4, §3.7, §4: all-LLM generation, answering, and judging remains the central validity threat

**Current claim:** The study treats 5-judge LLM panels as sufficient for the directional claim, with calibration and cross-provider agreement.

**Problem:** The entire pipeline is LLM-mediated: specifications are LLM-authored, batteries are LLM-generated from held-out passages, responses are LLM-generated, and scores are LLM-judged. Cross-provider LLM agreement helps, but it does not establish construct validity for “representational accuracy.” The paper acknowledges this, but for a top venue this is not merely a limitation; it is a core threat to the empirical claim.

**Suggested fix:** Add a human validation subset. At minimum, validate:
- a stratified set of questions for leakage, false premise, and answerability;
- a stratified set of responses for rubric correctness;
- whether the headline C5→C4a and C5→C2a improvements are preserved under human scoring.

If human validation is impossible before submission, the paper should be framed explicitly as a **methodological prototype with LLM-judge-only evidence**, not as an established empirical result.

---

### 2. §4.1: the headline gradient is vulnerable to delta-vs-baseline coupling

**Current claim:** “The less the model already knows about a person from pretraining, the more the specification helps,” supported by regressing Δ_C4a = C4a − C5 on C5, slope −0.96, R² = 0.82.

**Problem:** Regressing a change score on its baseline component mechanically induces negative association, especially on a bounded 1–5 scale with floor/ceiling effects. If C4a scores occupy a narrower band than C5, a near −1 slope can emerge partly by construction. The new battery-composition sensitivity block is useful, but it does not address this statistical coupling.

**Suggested fix:** Add analyses that separate genuine moderation by baseline from arithmetic coupling:

- Regress **C4a level** on C5, not only Δ on C5.
- Report whether low-baseline subjects improve more than expected under a null where C4a is independent of C5 but bounded.
- Use permutation or simulation baselines preserving the marginal distributions of C5 and C4a.
- Consider ANCOVA-style models: C4a ~ C5 + condition/battery covariates.
- Report confidence intervals via subject-level bootstrap, not only parametric regression.
- Tone down “baseline explains 82% of variance in spec effect” unless the coupling analysis shows this is not mostly mathematical.

The gradient is plausible and visually likely real, but the current statistical framing overstates what the slope proves.

### 7. §3.4, Appendix B: question validity needs stronger audit than n-gram leakage

- Is the question answerable from training-half patterns alone?
- Is the ground-truth passage actually a behavioral sample rather than a literal fact?
- Does the question contain false premise or overly leading wording?

## Comparison to the field

### PersonaGym

PersonaGym evaluates whether models maintain a persona consistently. Beyond Recall is more ambitious and more behaviorally grounded: it asks whether a representation helps predict held-out behavior from a specific person’s own writing. That is a distinct and valuable axis. The paper should keep emphasizing that persona fidelity is not behavioral prediction.

### Twin-2K

Twin-2K is the closest prior work because it predicts held-out behavior for many individuals. Its strengths are scale and human-subject grounding. Beyond Recall’s distinct contribution is open-ended, cross-format prediction: autobiographical prose → structured behavioral specification → free-text prediction against held-out autobiographical passages. However, Twin-2K is much stronger on sample size and human-grounded measurement. Beyond Recall is stronger on representation interpretability and provenance.

### LongMemEval and LoCoMo

These are recall-oriented memory benchmarks. Beyond Recall’s contribution is to show that recall-oriented memory systems can be evaluated on a downstream behavioral-prediction axis. The paper’s critique of recall-only evaluation is convincing. The memory-system comparisons are useful, but they should be presented as task-specific evidence rather than definitive vendor benchmarking.

### LoCoMo

LoCoMo is long-conversation factual memory. Beyond Recall is not a replacement; it is orthogonal. The paper’s strongest field-level contribution is the argument that memory systems need multiple axes: recall, preference alignment, persona fidelity, and representational/behavioral prediction.

### Overall field position

Beyond Recall’s distinct contribution is:

> A prototype benchmark and representation pipeline for evaluating whether compact, inspectable behavioral representations improve open-ended prediction of a specific person’s held-out behavior.

That is novel and worth pursuing. The current paper is not yet a definitive benchmark paper because the evaluation stack is not sufficiently validated, but it is a strong methodological proposal with promising preliminary evidence.

## What I would push back on in peer review

If I were reviewing this for a top empirical ML/HCI venue, I would push back on five things.

### 1. “No humans in the loop” is not acceptable for the central empirical claim

The paper can use LLM judges, but it needs human validation. Without it, I would not accept the paper as a strong empirical result.

### 2. The gradient analysis currently overclaims

The baseline-gradient finding is plausible, but the slope/R² framing is vulnerable because the dependent variable contains the independent variable. I would ask for a stronger statistical treatment before accepting “baseline explains 82% of variance.”

### 3. The deployment claim is still extrapolative

The low-baseline historical slice is suggestive, not deployment evidence. I would require either a living-user replication or much more cautious language.

### 4. Question generation needs independent validation

Backward-designed LLM questions can leak answer structure semantically. The current leakage audit is insufficient.

### 5. Internal consistency must be fixed

The battery-generator inconsistency, response-model coverage overstatement, memory-system additivity wording, and appendix numerical mismatch all need correction. These are not fatal scientifically, but they undermine trust.
