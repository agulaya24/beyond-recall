# `figures/` — Publication figures for the "Beyond Recall" paper

These PNG/PDF figures are the paper's publication-quality figures, rendered at 300 DPI. The current draft is [`../docs/beyond_recall_v8_draft.md`](../docs/beyond_recall_v8_draft.md); it explicitly references the `fig_4_1_*`, `fig_4_2_*`, and `fig_4_2_1_*` figures by section. The earlier `fig1` – `fig11` set was produced for the v6 draft (archived at [`../docs/versions/beyond_recall_v6_draft.md`](../docs/versions/beyond_recall_v6_draft.md)) and is retained as prior-draft artifacts; some numbers in these earlier figures reflect the 7-judge aggregate rather than the v8 5-judge primary panel. **Archive candidate — flag for author decision:** consider moving `fig1_*` through `fig11_*` into a `figures/_archive/` subfolder so a reader opening the folder sees only the paper-bound set.

The set was revised on 2026-04-22 after a cross-LLM figure-quality review identified five caption/figure correctness mismatches and a colorblind-safety issue on 9 of 14 figures. The fix log for that pass is at [`../docs/reviews/figure_fixes_20260422.md`](../docs/reviews/figure_fixes_20260422.md).

For the canonical numbers behind every figure, see [`../docs/DATA_REFERENCE.md`](../docs/DATA_REFERENCE.md).

## How the figures are generated

| Script | Purpose |
|---|---|
| `generate_figures.py` | v1 — superseded. Kept for reference. |
| `generate_figures_v2.py` | v2 — superseded. Kept for reference. |
| `generate_figures_v3.py` | **Canonical.** Regenerates the 9 PNGs `fig1` through `fig9`. The 14-subject baseline and spec arrays are inlined at the top of the script (sourced from the harmonized rerun `summary.json`). Uses `seaborn.set_theme('colorblind')` for a unified colorblind-safe palette, paper-scale typography, and shape redundancy on every color-encoded comparison. Run: `python generate_figures_v3.py` — writes PNGs next to the script. |
| `../scripts/generate_fig10_letta_scaling.py` | Renders `fig10_letta_scaling.png`. Run: `python scripts/generate_fig10_letta_scaling.py`. |
| `../scripts/generate_fig11_tier2_replication.py` | Renders `fig11_tier2_replication.png`. Run: `python scripts/generate_fig11_tier2_replication.py`. |
| `../scripts/generate_fig_4_1_gradient_scatter.py` | Renders `fig_4_1_gradient_scatter.{png,pdf}`. |
| `../scripts/generate_fig_4_2_compression.py` | Renders `fig_4_2_compression.{png,pdf}`. |
| `../scripts/generate_fig_4_2_1.py` | Renders `fig_4_2_1_question_improvement_rates.{png,pdf}`. Counts are inlined from the §4.2.1 table in the v8 draft. |
| `../scripts/generate_fig_4_4_1_jaccard_heatmap.py` | Renders `fig_4_4_1_jaccard_heatmap_v1.{png,pdf}`. Reads per-pair Jaccards from `docs/research/retrieval_overlap_analysis_20260501.json`. |

If you update the underlying scores, edit the inlined arrays in `generate_figures_v3.py` (and the corresponding v8-era scripts) to match `../docs/DATA_REFERENCE.md` and rerun. The generator scripts do not re-read raw results files; they are the rendering layer, not the analysis layer.
